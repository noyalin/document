Initial Validation - Overview

Details on the initial validation procedure

Introduction

Initial API validation is done by way of the online API validator tool: a utility that runs a series of tests against your Voyageone OpenAPI and provides real-time feedback to partners about any endpoints that display behavior or return responses that are not consistent with the API requirements as outlined in the Endpoints section of this documentation. This allows your developers to verify basic API functionality, ensure proper payload structure, and detect bugs in your API endpoints.

What the validator tool tests

Validation is divided into two test suites-- one suite for orders endpoints, and one for products (see below). Each suite contains a certain number of tests. Each suite can be run independently, allowing partners to develop and test the orders and products endpoints concurrently. However, each test within a suite relies on the data and successful outcome of the preceding test of that suite in order to run. For this reason, the test suite terminates once the first validation failure is detected.

What the validator tool does not test

Although the validator tool does verify some basic functionality of the API endpoints, it is not comprehensive. For example, it does not attempt to test:

  • Pending status orders
  • Partial cancellations
  • Partial refunds
  • Partial shipments

Additionally, the validator tool is currently configured only for standard OpenAPI implementations. If any of the following are true about your API, you may not be able to complete a successful validator run of one or both test suites. If this is the case, please email your technical contact.

  • Cannot return async responses within a few minutes
  • Does not support cancellations
  • Does not support refunds
  • Does not support shipments
  • Requires products fields other than those defined on the as defined on the Product Fields page
  • Has strict data requirements on product fields

We are constantly working to make the API development process as easy as possible. In the future, we hope to expand the validator to verify more endpoints and situations, as well as to add more configuration options to handle channels with special OpenAPI implementations as outlined above.

Validation test results

At the end of each test suite run, the result of the test suite is displayed, indicating the status of each test. If the test failed, there will also be a message indicating which aspect of the API failed to pass validation. Logs of the validation steps and HTTP traffic are provided for each test. A link is also provided to download a combined log of all tests.

If any validation run is unclear to you, or if you experience any problems with the API validator, please email your technical contact with your question and the URL of the validation run.

Next steps

After successful validation with this online validator tool, Voyageone will begin the next phase of the partner launch cycle: integration verification. That will include more in-depth validation of the API than is currently possible in this online validator tool. It will also include the creation and testing of an synchronization task that will provide the integrate between Voyageone's test environments and the OpenAPI.

Example mode

Running a test suite in "Example mode" instructs the validator to return what a successful run against the API should look like. It does this by running a test suite against a test API component implemented by Voyageone that passes all validation requirements. Use this example validation feature as a model or "cheat sheet" when writing or debugging your OpenAPI.

Products test suite

The products test suite runs a series of tests against the products endpoints of the OpenAPI and provides real-time feedback to partners about any endpoints that display behavior or return responses that are not consistent with the API requirements as outlined in the Products API Endpoints documentation.

The test suite contains the following tests:

# Name Description
1 Add products Creates a certain number of new products on the API
2 Get all products Gets all products from the API and validates their state
3 Add update products Creates and updates a certain number of products on the API
4 Get all products Gets all products from the API and validates their state
5 Update price and quantity Updates the price and quantity of certain items on the API
6 Get all products Gets all products from the API and validates their state
7 Update listing status Updates the listing status of certain items on the API
8 Get all products Gets all products from the API and validates their state
