Response Format

Outlines the acceptable formats of API responses.

All response payloads from the API should be in the form of a JSON-serialized ActionResponse. At the root level, this model contains several key properties:

  • a status indicating the over all success or failure of the request as a whole
  • a collection of Error objects to report any response-level errors
  • a string to return a batch response URL (if the response is status pending)
  • the actual data of the response (an object or a list of objects)

If the response is pending, or if a response-level error occurs, the ResponseBody property of the payload may be returned as null since there is no data to return in these cases. However, all successful, non-pending responses must return a valid non-null object matching the type detailed in the appropriate endpoint specification.
