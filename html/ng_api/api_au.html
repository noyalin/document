Authentication

Describes the API authentication protocols.

Seller identification

When on-boarding new sellers, you (the integration partner) will provide each seller with a unique seller ID and seller API token. The seller will then provide these identifiers in the Voyageone application when they configure their integration. Voyageone will include them in every request to your API in the form of the SellerID and SellerToken HTTP headers. You should use these header values to associate the request with a seller's account and to authenticate the integration.

HTTP Header Description
SellerID A unique, seller-specific identifier assigned by your system.
SellerToken A unique, seller-specific token assigned by your system.

In the case of authorization failure, the API response should return one of the following ErrorID values:

Error ID Example Error Message
4002 (InvalidSellerID) Authorization failed. Invalid SellerID
4001 (InvalidToken) Authorization failed. Invalid SellerToken

Security Protocol

All connections from the Voyageone production environment must be made using HTTPS to provide transport layer security. It is strongly recommended that you also enable HTTPS transport for any testing or sandbox API as well.
