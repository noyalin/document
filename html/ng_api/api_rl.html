Rate Limiting

options for rate limiting requests from Voyageone.

Voyageone will adhere to the rate limits agreed upon on the site configuration specification. It is recommended that the API also enforce similar rate limits to ensure that both systems are operating within the expected parameters. Some considerations to include when deciding on a rate limit:

  • Number of products per account
  • Each seller will be restricted to a predefined number of products they can list on your site.
  • The API rate limits should be calculated such that each product could be updated within a reasonable window. Excessive rate limiting can hinder a seller’s ability to manage multiple site listings.
  • Rate limit for API requests per account
  • Support for rate limiting requests per time interval will be supported. This can be defined at a seller or integration level. Example, 1000 requests per minute per seller.
  • SLAs for responses to API requests
Should the rate limit be exceeded, the API response should return the following Error:

Error ID Example Error Message
3002 (RateLimitExceeded) Rate limit exceeded. Please throttle your requests
